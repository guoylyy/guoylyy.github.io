Plan of Github 100 Streaks

2015-09-23

说明

本计划是实现个人技术一次长期性提升而制定的计划，旨在帮助开发人员养成编码的良好习惯 、提升编码热情、锻炼编码技艺。 因此，本计划参与人员必须严格遵守一下规定，否则计划终止。

规定

powered by 177days of github

他提出的原则有：

  1. Edits to README don’t count unless significant in nature.
  2. No scripting; you must personally make the contributions.
  3. No funny business editing commit times and such.

这里总结和补充一下原则：

  1. 必须是有意义的提交，feature or fix，feature可以拆分，但是必须完整，可以测试。

  2. 不能隔天提交，中断即挑战失败；

  3. 不限制维护的项目，但是每一个项目必须有良好的 readme 说明，并且定期更新；

  4. 每日提交后记录自己提交的内容，记录在 commit 信息里，需要详细可阅读；

  5. 定期和伙伴 code review；

  6. 维护的代码持续集成、自动测试、报告、周知。

你将获得

  • 持续编码的良好习惯；
  • 维护一个完整项目的经历；
  • 代码编码能力和习惯（code review）

总之，如果和我一起完成了这项挑战，你一定是一个优秀的程序员。
